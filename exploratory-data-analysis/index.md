# This project is in development

EDA is referenced throughout the portfolio as a consistent practice, but there is no project where it is the primary deliverable. Every current page uses EDA as a means to an end — to inform a model or test a hypothesis. A dedicated EDA project would be the one place where exploration itself is the product, which is a genuinely different analytical mindset worth demonstrating.

ydata_profiling (formerly pandas-profiling) is well-suited to this because it generates a comprehensive automated profiling report — distributions, correlations, missing value analysis, duplicate detection, interaction plots — that makes for a compelling visual deliverable. It also gives an opportunity to demonstrate something important: the difference between automated profiling as a starting point and genuine analytical interpretation. The tool produces the report; the data scientist decides what it means and what to do about it.

For dataset choice, use something with real analytical texture — missing values, skewed distributions, mixed variable types, at least one interesting correlation story. The UCI Adult Income dataset or the Ames Housing dataset would both work well, offering enough variety to demonstrate the full range of EDA outputs without being unwieldy.

To be included in the portfolio in a broadened Data Quality & Validation category alongside Great Expectations. The two projects together would make a strong paired statement — one about profiling and understanding data, one about validating it systematically.
